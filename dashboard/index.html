Conductor

The first infrastructure on Monad that sees transactions ~1 second before they're finalized and analyzes them with AI in real-time.

Why Conductor?

Without Conductor

You're Always Late

  • You see transactions after they're finalized
  • A whale dump already moved the price
  • Your position got liquidated before you noticed
  • Sandwich attacks complete before you can react
No human can react in 2 seconds. By the time you check, it's already over.
With Conductor

1 Second Head Start

  • Capture blocks at Proposed stage via monadNewHeads
  • AI analyzes risk and assigns severity in real-time
  • Your agent gets structured alerts with suggested actions
  • Autonomous on-chain actions before block finalization
AI agents react in milliseconds. Hedge, add collateral, exit — all before finalization.

Monad's Pipelined Consensus (MonadBFT)

Monad uses pipelined BFT consensus with 400ms block times. Three blocks are in-flight concurrently. Conductor captures at the earliest stage — Proposed — giving agents maximum reaction time.

Proposed
~0ms · 60%
Voted
~400ms · 95%
Finalized
~800ms · 99.9%
Verified
~2000ms · 100%

Standard RPC fires at Voted. Conductor captures at Proposed — ~400ms earlier per block.

Connect Your Agent

Any AI agent connects to Conductor in 3 lines of code. Receive real-time risk intelligence, then your agent decides what to do.

Quick Start — 3 Lines

agent.ts
import { ConductorClient } from "./src/sdk"; const conductor = new ConductorClient(); conductor.on("alert", (alert) => { // Your AI agent receives every alert in real-time console.log(alert.severity); // "low" | "medium" | "high" | "critical" console.log(alert.aiReasoning); // AI's risk explanation console.log(alert.suggestedActions); // ["hedge", "add_collateral", ...] console.log(alert.commitState); // "proposed" | "voted" | "finalized" });

That's it. Your agent is now receiving real-time Monad risk intelligence.

Filter by Severity

Only receive alerts that matter to your strategy:

severity-filter.ts
const conductor = new ConductorClient({ minSeverity: "medium" }); conductor.on("critical", async (alert) => { // EMERGENCY: add collateral before liquidation await vault.addEmergencyCollateral(alert.txHash, alert.severity); }); conductor.on("high", async (alert) => { // HIGH RISK: consider hedging await evaluateHedge(alert); }); conductor.on("medium", (alert) => { // MODERATE: log and monitor logToDatabase(alert); });

Filter by Transaction Type

type-filter.ts
// React to swaps conductor.on("swap", (alert) => { if (alert.estimatedPriceImpact > 2.0) { console.log(`Large swap: ${alert.estimatedPriceImpact}% impact`); } }); // Track whale movements conductor.on("large_transfer", (alert) => { console.log(`Whale: $${alert.valueUsd}`); });

config.ts
const conductor = new ConductorClient({ url: "wss://conductor-production-b213.up.railway.app", autoReconnect: true, // Auto-reconnect on disconnect minSeverity: "medium", // Minimum severity filter types: ["swap"], // Only specific tx types });

Alert Format & API

Every alert is a structured JSON object with risk scoring, AI reasoning, and suggested actions.

Alert Object

{ "txHash": "0xca57dc...c546c5", "blockNumber": "21754600", "commitState": "proposed", // proposed | voted | finalized "alertConfidence": "60%", // confidence based on commit state "severity": "medium", // low | medium | high | critical "type": "swap", // swap | large_transfer | contract_call "valueUsd": 12.50, "estimatedPriceImpact": 1.25, // percentage "affectedPool": "DEX Router", "suggestedActions": ["monitor", "check_exposure"], "aiReasoning": "Medium-value swap with 1.25% impact...", "confidence": 0.82, // AI confidence 0-1 "senderBalance": "45.2 MON", "capturedAt": 1711612800000, // when block was captured "analyzedAt": 1711612801200, // when analysis completed "latencyMs": 1200 // total pipeline latency }

CommitState & Confidence

Each alert carries the block's commit state, reflecting how far it is in Monad's consensus pipeline:

Proposed
60% confidence
Voted
95% confidence
Finalized
99.9% confidence
Verified
100%

Your agent can use commitState to decide how aggressively to act. At "proposed" stage (60%), you might monitor. At "voted" (95%), you act with confidence.

Dual-Layer AI Analysis

Layer 1: Rule-Based

~5ms latency, free
Low-severity transactions are analyzed instantly with deterministic rules. No API cost. Handles ~70% of all transactions.

Layer 2: GPT-4o-mini

~1-2s latency, AI-powered
Medium/high/critical severity gets full AI analysis with reasoning, confidence scoring, and suggested actions.

WebSocket Connection

Connect directly with any WebSocket client:

// Raw WebSocket (any language) const ws = new WebSocket("wss://conductor-production-b213.up.railway.app"); ws.onmessage = (event) => { const data = JSON.parse(event.data); if (data.type === "history") { // Array of recent alerts sent on connect console.log(data.alerts); } if (data.type === "alert") { // New real-time alert console.log(data.alert); } };

Works with any language that supports WebSocket — Python, Go, Rust, JavaScript, etc.
